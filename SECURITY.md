# Security Policy

## Supported versions

Only the latest version deployed at [unhookd.vercel.app](https://unhookd.vercel.app) receives security attention.

## Reporting a vulnerability

Please **do not** open a public GitHub issue for security vulnerabilities.

Email: luka.budik@rohlik.cz

Include:

- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (if any)

You'll receive a response within 48 hours. If confirmed, a fix will be deployed as quickly as possible and you'll be credited in the release notes (unless you prefer anonymity).

## Scope

**In scope:**

- Data leakage between anonymous users
- Firebase security rule bypasses
- XSS vulnerabilities
- Authentication issues

**Out of scope:**

- Bugs with no security impact
- Denial of service against the Vercel deployment
- Issues requiring physical access to the device
